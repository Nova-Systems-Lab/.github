# Support

Thank you for your interest in Nova Systems Lab projects.

Before asking for help, please check the project README, documentation, existing GitHub Discussions, and open or closed Issues. Your question may already have an answer.

## Where to Get Help

### Discord

For quick questions, community discussion, testing feedback, and contributor coordination, join the Nova Systems Lab Discord server:

[Join Nova Systems Lab on Discord](https://discord.gg/sfFyVyTfX8)

Discord is intended for informal discussion and coordination. Important technical decisions and project history should still be recorded on GitHub.

### GitHub Discussions

Use GitHub Discussions for:

- general project questions
- detailed feature proposals
- architecture and design discussions
- roadmap suggestions
- ideas that need public and searchable discussion
- contributor help

Please search existing discussions before opening a new one.

### GitHub Issues

Use GitHub Issues for:

- confirmed and reproducible bugs
- accepted feature work
- documentation problems
- performance issues
- clearly defined development tasks

Please do not use Issues for general questions or early-stage brainstorming.

When reporting a bug, include:

- the project name
- your Windows version
- the application or build version
- clear steps to reproduce the problem
- the expected and actual behaviour
- relevant screenshots or logs

Remove usernames, local paths, tokens, IP addresses, and other sensitive information before sharing logs.

### Pull Requests

Use Pull Requests to submit:

- code changes
- bug fixes
- tests
- documentation improvements
- build or configuration changes
- approved feature implementations

Whenever possible, link the Pull Request to an existing Issue.

Example:

```text
Closes #123
```

## Security Vulnerabilities

Do not report security vulnerabilities through public Discord channels, Discussions, or Issues.

Please follow the instructions in the project's SECURITY.md file and use GitHub's private vulnerability reporting feature where available.

## Response Expectations

Nova Systems Lab is an independent open-source organization, and contributors may be working in their free time.

We will try to respond when possible, but immediate support and fixed response times cannot be guaranteed. Please remain patient and respectful.

## Official Source of Truth

Discord is for conversation and coordination.

GitHub remains the official source of truth for:

* source code
* issues
* pull requests
* technical decisions
* documentation
* releases
* development history
